## Gallery-related files

| Path | Purpose |
| --- | --- |
| `src/components/gallery-section/GalleryCarousel.tsx` | Full-screen modal carousel view |
| `src/components/gallery-section/GalleryMosaic.tsx`   | Grid / masonry layout inside modal |
| `src/components/gallery-section/GalleryPreview.tsx`  | 5-image preview grid on homepage |
| `src/components/gallery-section/index.tsx`           | Section wrapper rendered on home page |
| `src/config/galleryConfig.ts`                        | Category labels & keyword mapping |
| `src/utils/galleryConfigUtils.ts`                    | Helpers to read ordering files, etc. |

All gallery UI now lives in **one folder** (`gallery-section`) keeping concerns co-located while business/config logic stays in their respective folders.

---

Future conventions:

* New cross-section UI ⇒ **components/** sub-folder per feature.
* Pure data/constants ⇒ **config/**.
* Pure functions/helpers ⇒ **utils/**.
* Avoid duplicate tooling at repo root; each app has its own build setup or live in an `apps/` monorepo layout.
